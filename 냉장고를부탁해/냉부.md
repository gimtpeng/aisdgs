# Project: SDGs Goal 12 - Zero Waste AI Refrigerator Dashboard

## 1. 개요 및 목적
- **목적:** 사용자의 음식 취향과 선호를 분석하고, 냉장고에 있는 유통기한이 임박한 식재료를 조합하여 맞춤형 '제로 웨이스트(Zero Waste)' 레시피를 추천하는 Streamlit 애플리케이션.
- **SDGs 목표 연계:** 목표 12 (지속가능한 소비와 생산 패턴 보장) - 음식물 쓰레기 감소 및 탄소 발자국 저감 시각화.

## 2. 핵심 기능 요구사항

### 기능 A: 7~8문항의 음식 취향 정보 테스트 (최초 진입 화면)
- 최초 진입 시 `st.session_state`를 사용하여 취향 분석 단계를 먼저 노출 (완료 전까지 메인 대시보드 진입 제한).
- 사용자의 지루함을 방지하기 위해 프로그레스 바(`st.progress`)를 상단에 배치.
- **랜덤 및 고정 7~8개 질문 구성 예시 (라디오 버튼 또는 셀렉트박스 사용):**
  1. 평소 선호하는 요리 스타일은? (한식 / 중식 / 일식 / 양식 / 퓨전)
  2. 기피하거나 알레르기가 있는 식재료가 있나요? (없음 / 견과류 / 갑각류 / 유제품 / 비건 선호)
  3. 요리할 때 가장 중요하게 생각하는 것은? (조리 시간(초간단) / 영양 밸런스 / 비주얼 / 맛의 깊이)
  4. 평소 선호하는 매운맛 단계는? (백김치 수준 / 신라면 수준 / 불닭 수준)
  5. 선호하는 메인 재료 종류는? (육류 중심 / 해산물 중심 / 채소·두부 중심 / 탄수화물류 면·밥)
  6. 오늘 나의 요리 컨셉은? (홈파티 정취 요리 / 건강한 다이어트식 / 든든한 보양식)
  7. 선호하는 조리 방식은? (국물·탕·찌개 / 볶음·구이 / 샐러드·무침 / 에어프라이어·오븐)
  8. 디저트나 간식 배는 따로 있다? (네! 상큼한 디저트가 필요해요 / 아니요, 깔끔한 마무리로 충분해요)
- '취향 분석 완료' 버튼을 누르면 결과 스코어가 세션에 저장되고 메인 화면으로 전환.

### 기능 B: 사이드바 (식재료 입력 및 냉장고 관리)
- 식재료명(`st.text_input`), 유통기한(`st.date_input`), 수량(`st.number_input`) 입력 필드 제공.
- '냉장고에 넣기' 버튼 클릭 시 하단 또는 세션 리스트 데이터프레임에 추가.

### 기능 C: 메인 대시보드 (상단 - 현황판)
- **유통기한 임박 알림:** 오늘 날짜 기준으로 유통기한이 3일 이내로 남은 식재료가 있다면 `st.warning` 또는 `st.error`로 시각적 경고 처리.
- **Zero Waste 에코 메트릭 (`st.metric`):** 
  - 이번 달 구출한 식재료 개수 (개)
  - 절약한 가계 비용 (원)
  - 저감한 탄소 배출량 ($CO_2$ kg) -> 약식 계산식 적용.

### 기능 D: AI 맞춤형 레시피 추천 (하단)
- 유저가 현재 냉장고에 등록한 재료 중 요리에 쓸 재료들을 `st.multiselect`로 다중 선택.
- '레시피 추천받기' 버튼 클릭 시 **[선택한 재료 + 정보 테스트에서 파악한 7~8개 취향 결과]**를 매칭하여 최적의 레시피 2~3개를 추천.
- *참고:* 초기 구현을 위해 딕셔너리 구조 of Mock Data(예시 레시피 셋)를 풍부하게 생성하여, 조건 필터링이 정교하게 작동하도록 구현할 것.
- 레시피는 `st.container`와 마크다운 카드를 사용하여 요리명, 조리시간, 난이도, Zero Waste 팁을 깔끔하게 출력.

## 3. UI/UX 디자인 가이드
- **테마 컬러:** 지구나 환경을 상징하는 그린(#2E7D32)과 에코 내추럴 톤을 메인 포인트 컬러로 사용.
- **레이아웃:** 깔끔한 화면 분할과 이모지 적극 사용을 통해 친근하고 귀여운 톤 앤 매너 구현.
- **상태 관리:** 화면이 리프레시되더라도 입력한 냉장고 데이터와 유저 취향 점수가 날아가지 않도록 `st.session_state`를 철저하게 설계.
